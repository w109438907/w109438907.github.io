总结

MVP 的目的是降低耦合度，分离 Model 层和 View 层，添加 Presenter 层作为中间件。

View 层 : 主要负责UI以及用户交互。Android中体现为Activity 和 Fragment 。

Model 层 : 负责数据相关的操作。

Presenter 层 : 连接 View 层与 Model 层，主要负责处理业务逻辑部分。

Dagger2 通过Java注解在编译期来实现依赖注入，既能达到解耦的目的，又能简化代码。
